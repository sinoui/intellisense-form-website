## 2. 平滑升级

可以从任何版本平滑升级到 v1.15.0-alpha.7。

## 3. sinomatrix 依赖变更

因 sinomatrix 缺陷影响了智能表单之前版本（v1.12.0 版本之前的版本）提供的功能，sinomatrix 已经修复，所以需要同步升级 sinomatrix 到 [1.6.37](http://47.93.34.153:10080/sino-matrix/sino-matrix/blob/master/CHANGELOG.md) 。（注意：访问此地址时可能会出现 ERR_UNSAFE_PORT 的情况，请参看[这篇帖子](https://blog.csdn.net/kjcxmx/article/details/118122483)配置一下浏览器。）

## 4. 主要变更

- fix: 修复表单保存时因请求中缺少 workflowid 导致失败的缺陷
- fix: 修复查询列表打不开的缺陷
- fix: 起草时保存多次，一直调用 post 接口的缺陷
- fix: 修复默认查询条件为空或者解析失败导致查询页打开失败的缺陷
- fix: 修复组件重绘产生新的默认值后，导致表单状态被重置的缺陷
- fix: 导入表单与已存在表单都不存在时，才认为是新表
- fix: 应用导出时，listpageid为空时，自动补充
- fix: 应用导入不需要弹出资源选择框
- fix: 修复导入应用时，没有处理应用中的分组的问题
- fix: 后端业务数据保存失败，前端却提示保存成功
- fix: 修复导入应用或表单时，由于流程id错误，导致的发布流程失败问题
- fix: 修复保存表单数据时因缺少workflowid导致失败的缺陷
- fix: 无上级部门的情况，无流程表单打开报错
- chore: 统一日期选择、时间选择、日期时间选择的列尺
